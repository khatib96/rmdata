# قسم الضرائب والكيانات الضريبية — شرح تفصيلي

هذا المستند يشرح للمبرمج أو أداة الذكاء الاصطناعي كيف يعمل **قسم الكيانات الضريبية** (الضرائب): الكيان القانوني، TRN، ضريبة القيمة المضافة، ضريبة الشركات، ربط الأفرع بالكيان، المدفوعات، والمستندات.

---

## 1. نظرة عامة

قسم الضرائب يدير **الكيانات القانونية (Legal Entity)** التي تخضع للضرائب في الدولة:
- **TRN (Tax Registration Number):** رقم تسجيل ضريبي — 15 رقمًا.
- **ضريبة القيمة المضافة (VAT 5%):** دورة تقديم (ربع سنوي/شهري)، تواريخ تسجيل، ومدفوعات مرتبطة بفترات (ربع سنة أو شهر).
- **ضريبة الشركات (Corporate Tax 9%):** رقم تسجيل/جيبان، تاريخ نهاية السنة المالية، ومدفوعات سنوية.
- **ربط الأفرع بالكيان:** جدول `tax_entity_branches` يربط كل كيان بعدة أفرع؛ كل فرع لا يمكن أن يكون مرتبطاً بأكثر من كيان واحد.
- **المستندات:** شهادات ضريبة قيمة مضافة وشهادات ضريبة شركات (أقسام `vat_cert`, `corporate_tax_cert`).

## 2. الجداول والعلاقات

### جدول الكيانات: `entities`

| العمود | النوع | الوصف |
|--------|--------|--------|
| `id` | INTEGER PK | المعرف |
| `entityNickname` | VARCHAR(200) | لقب الكيان (مثل: أبوظبي وفروعها) |
| `name` | VARCHAR(200) | الاسم القانوني (يُستخرج غالباً من الفرع الرئيسي) |
| `nameEn` | VARCHAR(200) | الاسم بالإنجليزية |
| `registeredAddress` | TEXT | العنوان المسجّل |
| `contactNumber` | VARCHAR(50) | رقم الاتصال |
| `tradeLicenseNumber` | VARCHAR(50) | (إن وُجد) |
| `tradeLicenseExpiry` | DATE | (إن وُجد) |
| `trn` | VARCHAR(15) | Tax Registration Number — 15 رقم |
| `vatTrn` | VARCHAR(50) | (إن وُجد) |
| `vatRegDate` | DATE | تاريخ بدء ضريبة القيمة المضافة |
| `vatFilingCycle` | VARCHAR(50) | quarterly \| monthly — دورة تقديم VAT |
| `corporateTaxGiban` | VARCHAR(50) | رقم جيبان/تسجيل ضريبة الشركات |
| `corporateTaxTrn` | VARCHAR(50) | (إن وُجد) |
| `corporateTaxRegDate` | DATE | تاريخ تسجيل ضريبة الشركات |
| `financialYearEnd` | VARCHAR(20) | نهاية السنة المالية (مثل 12-31، 03-31) |
| `mainBranchId` | INTEGER | الفرع الرئيسي — يُستخدم لاستخراج الاسم القانوني من رخصة الفرع |
| `status` | VARCHAR(50) | active \| archived |
| `notes` | TEXT | |
| `createdAt`, `updatedAt` | DATETIME | |

**الكيان في TypeORM:** `src/database/entities/Entity.ts` — الاسم الداخلي للكلاس: `LegalEntity`.

### جدول ربط الكيانات بالأفرع: `tax_entity_branches`

| العمود | الوصف |
|--------|--------|
| `entityId` | FK → entities(id) ON DELETE CASCADE |
| `branchId` | FK → branches(id) ON DELETE CASCADE |
| **PRIMARY KEY** | (entityId, branchId) |

- **قاعدة العمل:** كل `branchId` لا يظهر إلا مع كيان واحد؛ أي فرع لا يمكن ربطه بأكثر من كيان ضريبي واحد.
- عند **التعديل** على الكيان: يُحذف كل الصفوف الخاصة به ثم يُعاد إدراج الفروع المختارة (بما فيها الفرع الرئيسي).
- في واجهة **الأفرع**: يمكن تصفية القائمة حسب "الكيان الضريبي" — أي عرض الأفرع التي لها سجل في `tax_entity_branches` لـ `entityId` المحدد.
- في **بروفايل الفرع**: يُعرض إن وُجد الكيان المرتبط (TRN، ضريبة شركات، إلخ) عبر استعلام `tax_entity_branches` ثم `entities`.

### جدول مدفوعات الضريبة: `tax_payments`

| العمود | الوصف |
|--------|--------|
| `id` | PK |
| `entityId` | FK → entities(id) ON DELETE CASCADE |
| `type` | 'vat' \| 'corporate' |
| `financialYear` | سنة مالية |
| `quarter` | للـ VAT فقط: 1–4 (ربع سنوي) |
| `periodFrom` | بداية الفترة (نص) |
| `periodTo` | نهاية الفترة (نص) |
| `amount` | المبلغ |
| `paymentDate` | تاريخ الدفع |

- **VAT:** يُخزَّن مع `quarter`, `periodFrom`, `periodTo`؛ المسار النسبي للمرفقات: `Taxes/{entityId}/payments/...`.
- **ضريبة الشركات:** بدون quarter/period؛ مدفوعة سنوياً حسب السنة المالية.

## 3. واجهات المستخدم

### قائمة الكيانات الضريبية (`Entities.tsx`)

- عرض الكيانات النشطة مع: لقب الكيان، TRN، حالة ضريبة الشركات (إن وُجد)، و**عدد الفروع المرتبطة** من استعلام فرعي على `tax_entity_branches`.
- زر إضافة يفتح `AddEntityModal`؛ الدخول لبروفايل كيان يوجّه إلى `EntityProfile` مع `id`.

### إضافة/تعديل كيان (`AddEntityModal.tsx`)

- **الحقول الإلزامية:** لقب الكيان، **الفرع الرئيسي** (مطلوب لاستخراج الاسم القانوني).
- **الفرع الرئيسي:** قائمة من الأفرع غير المؤرشفة؛ عند الاختيار يُجلب من `branch_licenses` الاسم التجاري (والإنجليزي) ويُملأ حقل الاسم القانوني (واسم En).
- **TRN:** اختياري؛ إن وُجد يجب أن يكون 15 رقمًا (التحقق: `/^\d{15}$/` بعد إزالة المسافات).
- **VAT:** تاريخ بدء السريان، دورة التقديم (ربع سنوي / شهري).
- **ضريبة الشركات:** رقم التسجيل/جيبان، تاريخ بدء السريان، نهاية السنة المالية (مثل 31 مارس، 30 يونيو، …).
- **الفروع المرتبطة (ربط الضريبة):**
  - في **وضع الإضافة:** تُعرض الأفرع التي **ليست** مرتبطة بأي كيان (`branchId` غير موجود في `tax_entity_branches`).
  - في **وضع التعديل:** تُعرض الأفرع التي إما غير مرتبطة بأي كيان أو مرتبطة **بهذا الكيان فقط** (لإمكانية الإبقاء أو الإضافة دون ربط فرع بكيانين).
- الفرع الرئيسي يُضاف تلقائياً إلى قائمة الفروع المرتبطة ولا يمكن إزالته من الربط.
- عند الحفظ:
  - **تعديل:** UPDATE على `entities` ثم DELETE من `tax_entity_branches` WHERE entityId، ثم INSERT لجميع الفروع المختارة (بما فيها الرئيسي).
  - **إضافة:** INSERT في `entities` ثم INSERT في `tax_entity_branches` لكل فرع مختار.
- **المستندات:** رفع شهادة VAT و/أو شهادة ضريبة شركات (أقسام `vat_cert`, `corporate_tax_cert`) مع حفظ عبر نظام المستندات.

### بروفايل الكيان (`EntityProfile.tsx`)

- **تبويبات:** المعلومات الأساسية، الفروع المرتبطة، ضريبة القيمة المضافة (5%)، ضريبة الشركات (9%)، مدفوعات الضريبة (ملخص)، المستندات.
- **المعلومات الأساسية:** عرض كل حقول الكيان + زر تعديل يفتح `AddEntityModal` مع `editEntityId`.
- **الفروع المرتبطة:** جدول يعرض لكل فرع: الاسم، الاسم التجاري، رقم الرخصة، وكونه "الفرع الرئيسي" أم لا؛ البيانات تُجمع من `branches` و `branch_licenses` حسب `branchId` القادم من `tax_entity_branches` و `mainBranchId`.
- **ضريبة القيمة المضافة:** تبويب `VatPaymentsTab` — إضافة/تعديل مدفوعات VAT (سنة، ربع، فترة، مبلغ، تاريخ، مرفق اختياري).
- **ضريبة الشركات:** تبويب `CorporatePaymentsTab` — إضافة/تعديل مدفوعات ضريبة الشركات (سنة مالية، مبلغ، تاريخ، مرفق اختياري).
- **ملخص مدفوعات الضريبة:** `TaxSummaryTab` — عرض موحّد لمدفوعات VAT والشركات.
- **المستندات:** `EntityDocumentsTab` — عرض ورفع وحذف مستندات أقسام `vat_cert` و `corporate_tax_cert`.
- **أرشفة:** تحديث `entities.status` إلى `archived` وتسجيل في `activity_logs`.
- **حذف نهائي:** حذف مرتب: أولاً `tax_payments`، ثم `tax_entity_branches`، ثم `entities`؛ لا حذف تلقائي للفروع نفسها.

### الربط مع الأفرع

- **في قائمة الأفرع (`Branches.tsx`):** فلتر "حسب الكيان الضريبي" — استعلام: الأفرع التي `id IN (SELECT branchId FROM tax_entity_branches WHERE entityId = ?)`.
- **في بروفايل الفرع (`BranchProfile.tsx`):** استعلام `entityId` من `tax_entity_branches` WHERE `branchId`؛ إن وُجد يُعرض TRN وحقل ضريبة الشركات (جيبان) من جدول `entities`.

---

## 4. الثوابت والقيم الثابتة

- **دورة VAT (VAT_CYCLES):** quarterly → ربع سنوي، monthly → شهري.
- **نهاية السنة المالية (FYE_MONTHS):** قيم مثل 03-31، 06-30، 09-30، 12-31 مع تسميات عربية.
- **أرباع السنة (QUARTERS):** Q1–Q4 مع `periodFrom` و `periodTo` لكل ربع (محددة في المكوّن).

---

## 5. المستندات

- **نوع الكيان في نظام المستندات:** `entity`.
- **الأقسام المستخدمة:** `vat_cert` (شهادات ضريبة القيمة المضافة)، `corporate_tax_cert` (شهادات ضريبة الشركات).
- المسار النسبي يتبع نمط مثل: `Taxes/{entityId}/...` أو ضمن مجلد الكيان.
