# سجل تنفيذ V1.2 — ربط المستخدمين بالموظفين وأصحاب العمل

هذا الملف يوثق كل التعديلات التي تم تنفيذها وفق خطة V1.2 (ربط المستخدمين بالموظفين وأصحاب العمل). لا يُعدّل ملف الخطة نفسها.

---

## 1. قاعدة البيانات (Migrations)

**الملف:** `electron/database/migrations.ts`

- **جدول `users`:**
  - `userType` (TEXT DEFAULT 'free')
  - `linkedEntityType` (TEXT)
  - `linkedEntityId` (INTEGER)
  - `mustChangePassword` (INTEGER DEFAULT 0)
  - `passwordChangedAt` (TEXT)
  - تحديث المستخدمين الحاليين: `UPDATE users SET userType = 'free' WHERE userType IS NULL`

- **جدول `employees`:**
  - `userId` (INTEGER REFERENCES users(id))
  - فهرس UNIQUE: `idx_employees_userId` WHERE userId IS NOT NULL

- **جدول `employers`:**
  - `userId` (INTEGER REFERENCES users(id))
  - فهرس UNIQUE: `idx_employers_userId` WHERE userId IS NOT NULL

- **فهرس على `users`:**
  - `idx_users_linked` UNIQUE على (linkedEntityType, linkedEntityId) WHERE linkedEntityId IS NOT NULL

---

## 2. معالجات IPC في Main Process

**الملف:** `electron/main.ts`

| المعالج | التعديل |
|---------|---------|
| **auth:login** | إرجاع حقول الربط: userType, linkedEntityType, linkedEntityId, mustChangePassword، وبيانات الربط من employees/employers: linkedEntityName, linkedEntityImagePath, linkedBranchName, linkedProfession |
| **auth:createUser** | دعم userType, linkedEntityType, linkedEntityId, mustChangePassword؛ التحقق من عدم ربط الكيان مسبقاً؛ تحديث employees.userId أو employers.userId بعد الإنشاء |
| **auth:setPassword** | تعيين mustChangePassword = 1 عند إعادة التعيين من المدير |
| **auth:updateUser** | دعم userType, linkedEntityType, linkedEntityId؛ فك ربط الكيان القديم وربط الجديد في employees/employers |
| **auth:deleteUser** | مسح userId من employees و employers قبل حذف المستخدم |
| **auth:searchLinkableEntities** | جديد — بحث في employees و employers حيث userId IS NULL، إرجاع قوائم مع code, name, branchName, profession |
| **auth:changeOwnPassword** | جديد — تغيير كلمة المرور للمستخدم الحالي + تعيين mustChangePassword = 0 و passwordChangedAt |

**الملفات:** `electron/preload.ts`, `src/types/electron.d.ts`, `src/types/shared.ts`

- **preload.ts:** تعريض `authSearchLinkableEntities(query)` و `authChangeOwnPassword(userId, currentPassword, newPassword)`؛ تحديث توقيعات `authCreateUser` و `authUpdateUser`.
- **electron.d.ts:** تحديث `AuthLoginResult` بحقول المستخدم الجديدة؛ إضافة أنواع دوال النسخ الاحتياطي والـ auth الجديدة.

  - `userType: 'free' | 'linked'`
  - `linkedEntityType?: 'employee' | 'employer'`
  - `linkedEntityId?: number`
  - `linkedEntityName?: string`
  - `linkedEntityImagePath?: string`
  - `linkedBranchName?: string`
  - `linkedProfession?: string`
  - `mustChangePassword: boolean`

## 4. واجهة إضافة المستخدم (AddUserModal)

**الملف:** `src/components/Settings/sections/UsersSettings.tsx`

- **الخطوة 1:** اختيار نوع المستخدم:
  - "مستخدم خارجي (حر)" — userType = free
  - "ربط بموظف / صاحب عمل" — userType = linked
- **الخطوة 2 — خارجي:** نموذج كامل: اسم مستخدم، اسم كامل، بريد، دور، كلمة مرور، تأكيد.
- **الخطوة 2 — مرتبط:**
  - حقل بحث يستدعي `authSearchLinkableEntities`؛ عرض النتائج (اسم، كود، فرع، نوع: موظف/صاحب عمل).
  - عند الاختيار: اسم المستخدم = الكود (قابل للتعديل)، الاسم الكامل للقراءة فقط، كلمة مرور مؤقتة، mustChangePassword = true.
- معالجة أخطاء: USERNAME_EXISTS, ENTITY_ALREADY_LINKED, ENTITY_NOT_FOUND.

## 6. تغيير كلمة المرور الإلزامي عند أول دخول

**الملف:** `src/pages/Login.tsx`

- بعد نجاح تسجيل الدخول: إذا `user.mustChangePassword === true` لا يتم التوجيه إلى `/dashboard`؛ يُعرض modal إلزامي.
- حقول الـ modal: كلمة المرور الحالية، كلمة المرور الجديدة، تأكيد كلمة المرور.
- عند النجاح: استدعاء `authChangeOwnPassword` ثم تحديث الحالة `mustChangePassword: false` في الـ store والتوجيه إلى `/dashboard`.
- تعبئة كلمة المرور الحالية تلقائياً من حقل تسجيل الدخول لتجنب إدخالها مرة ثانية.
