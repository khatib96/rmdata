# مقترح قسم السكن — توثيق وحدات السكن والدفعات والسكان

وثيقة مقترحات لتطوير قسم **السكن** في نظام إدارة الرداء الموحد، مع المحافظة على الشكل والهيكلة الموحدة مع الأفرع والمركبات.

---

## 1. الوضع الحالي في البرنامج

- جدول **`housing_units`** موجود ويحتوي:
  - اسم، نوع سكن، إمارة
  - بيانات عقد: رقم عقد، تاريخ إصدار، انتهاء، قيمة إيجار، طريقة دفع، عدد الأقساط
  - **باسم من**: شركة / موظف
  - ربط بفرع وموظف
- جدول **`housing_installments`** موجود للأقساط (استحقاق، مبلغ، مدفوع، تاريخ دفع) — نفس فكرة عقد الإيجار في الأفرع.

المنطق قريب من نظام الأفرع والمركبات؛ المطلوب توسيعه وواجهات وتجربة موحدة.

---

## 2. تعديلات مقترحة على نموذج البيانات

| المطلوب | الوضع الحالي | المقترح |
|--------|---------------|---------|
| **اسم المالك/المؤجر** | غير موجود | إضافة حقل `landlordName` (أو `ownerName`) في `housing_units`. |
| **نوع الوحدة + "باسم من"** | `housingType` (labour, family, warehouse, mixed) و `ownedBy` (company, employee) | الإبقاء على الحقلين مع توضيح التسميات في الواجهة: **نوع الاستخدام**: سكن عمال، سكن شخصي/عائلي، مستودع، مختلط. **العقد باسم**: الشركة، الموظف، طرف ثالث. ويمكن لاحقاً إضافة `contractInNameOf` = company \| employee \| other إذا أردت فصل "باسم من" عن "ملكية" الوحدة. |
| **المستأجر (للعرض والإدخال)** | لا يوجد حقل صريح | **المستأجر** = من يتحمل العقد أو من يُعرض اسمه في السجل. إما الاعتماد على الربط الحالي (فرع / موظف) كـ "مستأجر افتراضي"، أو إضافة حقل نصي `tenantDisplayName` (مثلاً "الشركة – فرع دبي"، أو اسم موظف، أو "سكن عمال – مبنى 3") لمرونة العرض دون تعقيد الربط. |
| **تعيين سكان** | لا يوجد | جدول جديد **`housing_occupants`**: (housing_unit_id, employee_id nullable, name nullable, role مثلاً "ساكن رئيسي", from_date, to_date). إذا كان الساكن موظفاً نربط بـ employee_id، وإلا نستخدم name فقط. |

## 3. واجهات وتدفق عمل موحد مع الأفرع والمركبات

### 3.1 قائمة وحدات السكن (صفحة السكن)

- **فلترة حسب**: الإمارة، نوع الوحدة، العقد باسم (شركة/موظف/آخر)، فرع مرتبط، حالة انتهاء العقد (قريب/منتهي).
- **عرض**: شبكة/جدول مع:
  - اسم الوحدة
  - نوع الاستخدام
  - العقد باسم
  - تاريخ انتهاء العقد
  - حالة الدفعات (مثلاً "3/12 مدفوعة")
  - عدد السكان إن وجد
- زر **إضافة وحدة سكن جديدة** يفتح نافذة بنفس أسلوب "إضافة مركبة" / "إضافة فرع".

### 3.2 نافذة إضافة/تعديل وحدة سكن

- **بيانات أساسية**:
  - اسم الوحدة
  - نوع الاستخدام: سكن عمال، سكن شخصي/عائلي، مستودع، مختلط
  - العقد باسم: الشركة، الموظف، طرف ثالث
  - إمارة، عنوان أو وصف إن احتجت

- **بيانات العقد**:
  - اسم المالك/المؤجر
  - رقم العقد
  - تاريخ الإصدار
  - تاريخ الانتهاء
  - قيمة العقد (الإيجار الكلي أو السنوي حسب ما تعتمد)
  - طريقة الدفع
  - عدد الأقساط

- **ربط اختياري**: فرع مرتبط، أو موظف (إذا العقد باسم موظف)، مع حقل **المستأجر (للعرض)** إن أضفناه.

- **نظام الدفعات**: نفس منطق الأفرع — إنشاء أقساط من (عدد الأقساط + تاريخ البداية/النهاية أو توزيع تلقائي)، مع تواريخ استحقاق ومبالغ، ووضع "مدفوع" + تاريخ الدفع. يمكن تبويب "الدفعات" داخل نفس النافذة أو في صفحة التفاصيل.

- **المستندات**: تحميل مستندات (عقد، إيصالات، تجديدات) مع تسمية واضحة، وربطها بالوحدة (نفس أسلوب المستندات في الأفرع/المركبات: entityType = housing, entityId = housing_unit_id).

### 3.3 صفحة تفاصيل وحدة السكن

- **تبويبات**:
  - المعلومات الأساسية
  - بيانات العقد والدفعات
  - السكان
  - المستندات
  - السجل (إن أضفتم سجلاً للعمليات لاحقاً)

- **تبويب السكان**: زر **تعيين سكان** (أو "إضافة ساكن") يفتح نافذة:
  - اختيار موظف من القائمة أو إدخال اسم يدوي
  - دور (ساكن رئيسي، إلخ)
  - تاريخ بداية ونهاية الإقامة
  - عرض جدول بالسكان الحاليين والسابقين مع التواريخ

- المحافظة على نفس الشكل العام: ألوان، تخطيط، أزرار، تبويبات.

## 4. تحسينات إضافية مقترحة

- **تنبيهات انتهاء العقد**: مثل المركبات والأفرع — تنبيه قبل انتهاء عقد السكن (مثلاً 30 أو 60 يوماً) وإظهاره في لوحة التنبيهات أو في الصفحة الرئيسية.

- **ربط مع الموظفين**: من بروفايل الموظف إمكانية عرض "وحدة السكن المرتبطة" (إن كان مرتبطاً كساكن أو كمستأجر) مع رابط لصفحة الوحدة.

- **ربط مع الأفرع**: في بروفايل الفرع عرض وحدات السكن المرتبطة به إن وُجدت (العلاقة موجودة حالياً عبر `branchId`).

- **تقارير بسيطة**: عدد الوحدات حسب النوع، وحدات بعقود منتهية قريباً، ملخص الدفعات (المدفوع / المتبقي) لكل وحدة أو إجمالي.

- **أكواد تلقائية**: مثل المركبات (RMV…) والأفرع (RMB…) استخدام كود لوحدات السكن (مثلاً RMH0001) لتسهيل الإشارة والبحث.

---

## 5. ترتيب تنفيذ مقترح

1. **قاعدة البيانات**: إضافة `landlordName` (وربما `tenantDisplayName`)، وإنشاء جدول `housing_occupants` مع العلاقات المناسبة.
2. **واجهة القائمة**: Housing.tsx — جلب الوحدات من DB، فلترة، عرض شبكة/جدول، زر إضافة.
3. **نافذة إضافة/تعديل وحدة**: نموذج كامل (بيانات أساسية + عقد + دفعات أولية + مستندات) مع الحفظ.
4. **صفحة التفاصيل**: HousingProfile مع التبويبات وزر تعيين سكان وتبويب الدفعات.
5. **تعيين السكان**: نافذة/مودال إضافة ساكن (موظف أو اسم) مع التواريخ وحفظها في `housing_occupants`.
6. **المستندات والسجل**: ربط نظام المستندات والسجل الحالي بـ housing بنفس أسلوب الأفرع/المركبات.

## 6. ملخص أنواع الوحدات (حسب فكرتك)

| النوع | الوصف | العقد باسم | المستأجر (للعرض) |
|------|--------|------------|-------------------|
| سكن عمال | سكن للعمال | شركة أو طرف ثالث | الشركة / الفرع |
| سكن شخصي | باسم الموظف والشركة مسؤولة | موظف أو شركة | الموظف |
| مستودع | عقد بيت لكن استخدام مستودع | شركة أو شخصي | حسب العقد |
| باسم الشركة | ملكية/إيجار باسم الشركة | شركة | الشركة / الفرع |

يمكن توسيع القائمة لاحقاً (مثلاً "سكن عائلي"، "غرفة مفردة") دون تغيير المنطق العام.
